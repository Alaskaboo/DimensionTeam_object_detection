# 模型说明

> 本章节解释模型来源、选择策略与落地建议。  
> 目标：让你在“速度、精度、稳定性”之间快速做出合适选择。

## 1. 模型来源结构

系统支持三类模型来源，统一在“高级模型选择”管理：

1. **本地资源模型**
2. **私有网络资源模型**
3. **官方网络资源模型**

你可以在同一界面完成浏览、下载、复制链接、查看详情与应用模型。

## 2. 本地资源模型

### 适用场景
- 已有训练好的 `.pt` 文件，需要快速验证
- 网络受限，不依赖在线下载

### 操作流程
1. 进入本地资源模型页
2. 设置/浏览模型目录
3. 刷新模型列表
4. 选中模型并应用

### 使用建议
- 模型文件名建议带版本号（如 `best_v3.pt`）
- 路径尽量稳定，便于复现测试
- 先用单图测试可用性，再进入视频/批量任务

## 3. 私有网络资源模型

### 适用场景
- 团队内统一分发模型
- 需要在线版本更新

### 操作流程
1. 设置下载目录（默认 `pt_models`）
2. 查看模型信息（大小、日期、类别数、状态）
3. 点击下载或复制链接外部下载
4. 下载完成后应用模型

### 状态说明
- `未下载`
- `下载中...`
- `已下载`

## 4. 官方网络资源模型

### 适用场景
- 用官方权重建立基线
- 对比速度与精度时需要标准参考

### 规格建议

| 规格 | 说明 | 推荐场景 |
|---|---|---|
| n (nano) | 最轻量 | 实时优先、低算力 |
| s (small) | 轻量平衡 | 日常验证 |
| m (medium) | 中等规模 | 通用桌面端 |
| l (large) | 大型模型 | 精度优先 |
| x (xlarge) | 超大型 | 高算力与上限测试 |

## 5. 模型选择策略（实用）

1. **先通后优**：先用 `n/s` 跑通，再切 `m/l/x` 提精度  
2. **固定阈值对比**：同素材 + 同阈值，结果更可信  
3. **按任务配模型**：  
   - 监控/视频：优先轻量模型保证流畅  
   - 批量离线：可选更大模型提升效果  
4. **导出复盘**：结合明细导出做量化评估

## 6. 注意事项

- 下载前确认目录可写、磁盘空间充足  
- 加载失败先检查文件完整性与权限  
- 模型更新后建议先做单图+短视频回归测试

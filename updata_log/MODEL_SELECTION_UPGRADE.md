# 🎯 增强版通用目标检测系统 - 模型选择功能升级

## 🚀 新增功能概述

本次升级为模型选择对话框增加了网络资源模型支持，采用Tab页设计，提供更直观的用户体验。

## 🔧 主要改进

### 1. Tab页界面设计
- **本地资源模型 Tab**：保留原有的本地模型管理功能
- **网络资源模型 Tab**：新增网络模型浏览和下载功能

### 2. 网络模型管理功能
- **模型展示**：显示模型名称、大小、修改时间、类别数量
- **状态跟踪**：实时显示下载状态（未下载/下载中/已下载/下载失败）
- **双击查看详情**：显示模型的完整类别信息弹窗
- **右键菜单**：提供便捷的下载操作选项
- **操作按钮栏**：每行包含下载和复制链接按钮，已下载模型的下载按钮自动置灰

### 3. 下载管理
- **默认路径**：模型自动下载到 `pt_model` 目录
- **自定义路径**：支持选择其他下载目录
- **重复检测**：智能检测已存在的模型文件
- **覆盖确认**：重复文件提供覆盖确认选项

## 💡 使用方法

### 1. 浏览网络模型
1. 打开模型选择对话框
2. 切换到"🌐 网络资源模型" Tab
3. 查看可用的网络模型列表

### 2. 下载网络模型
1. 右键点击想要下载的模型
2. 选择"📥 下载模型"选项
3. 或者双击模型查看详情后点击下载

### 3. 查看模型详情
1. 双击任意网络模型
2. 弹出详细信息窗口，包含：
   - 模型基本信息（名称、大小、修改时间）
   - 类别数量统计
   - 完整的类别名称列表

### 4. 使用操作按钮
1. **下载按钮**：点击下载模型，已下载的模型按钮自动变为灰色"✅ 已下载"
2. **复制链接按钮**：点击复制模型的GitHub下载链接到剪贴板
3. **状态同步**：下载完成后按钮状态自动更新

### 5. 选择模型
1. 在任一Tab中选择模型
2. 点击"确定"按钮完成选择
3. 系统会自动加载选中的模型

## ⚙️ 技术实现

### 核心组件
- **ModelSelectionDialog**：主对话框类
- **本地Tab管理**：继承原有功能
- **网络Tab管理**：新增网络模型处理逻辑

### 数据源
- **本地模型**：通过 `model_manager.scan_models()` 获取
- **网络模型**：从 `pt_files_report.csv` 文件读取

### 下载机制
- **URL构建**：`https://github.com/JingW-ui/Pl-MAPP/releases/download/pt_download/+权重文件名`
- **HTTP请求**：使用 `requests` 库进行文件下载
- **进度跟踪**：实时更新下载状态

## 🛠️ 依赖要求

```txt
# 新增依赖
requests>=2.25.0
```

## 📝 注意事项

1. **网络连接**：下载功能需要稳定的网络连接
2. **存储空间**：确保下载目录有足够的磁盘空间
3. **权限问题**：确保有写入目标目录的权限
4. **模型兼容性**：下载的模型需与当前YOLO版本兼容
